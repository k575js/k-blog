    # vue中的插槽

    # 一.插槽概念

    插槽（Slot）是 vue 为组件的封装者提供的能力。允许开发者在封装组件时，把不确定的、希望由用户指定的部分定义为插槽

    可以把插槽认为是组件封装期间，为用户预留的内容的占位符

    # 二.匿名插槽

    1. 在封装组件时，可以通过 <slot>元素定义插槽，从而为用户预留内容占位符
    2. 如果在定义slot的时候，没有为插槽<slot>指定name属性，则这个slot就是匿名插槽
    //封装组件
<template>
  <div class="btn">
    <slot></slot>
  </div>
</template>

    # 注意：

    如果在封装组件时没有预留任何 <slot>插槽，则用户提供的任何自定义内容都会被丢弃 封装组件时，可以为预留的<slot> 插槽提供后备内容（默认内容）。如果组件的使用者没有为插槽提供任何内容，则后备内容会生效

    # 三.具名插槽

    如果在封装组件时需要预留多个插槽节点，则需要为每个<slot> 插槽指定具体的name 名称 这种带有具体名称的插槽叫做“具名插槽”。

    //封装组件
<template>
  <div class="header">
    <div class="left">
      <slot name="left"></slot>
    </div>
    <div class="center">
      <slot name="center"></slot>
    </div>
    <div class="right">
      <slot name="right"></slot>
    </div>
  </div>
</template>

    2. 跟v-on 和v-bind 一样，v-slot也有缩写，即把参数之前的所有内容 (v-slot:) 替换为字符#。例如v-slot:header可以被重写为#header。

    2. 补充一下：vue2.0中也支持slot='名称'的写法

    # 四.作用域插槽

    # 1.作用域插槽的定义

    在封装组件的过程中，可以为预留的 <slot>插槽绑定插槽 props数据，这种带有插槽props 数据的<slot> 叫做“作用域插槽”。

    //封装组件
<template>
 	//作用域插槽:封装组件过程中,预留插槽的同时,用属性的形式,为slot提供了数据
	<span>
	 <slot v-bind:user="user">
	   {{ user.lastName }}
	 </slot>
	</span>
</template>

    # 2.作用域插槽的使用

    在使用包含作用域插槽的组件时，可以使用 v-slot: 插槽名称='包含所有插槽 prop 的对象'的形式，接收作用域插槽对外提供的数据。 简单来说,就是在父级作用域中，我们可以给 v-slot 带一个值来定义我们提供的插槽 prop 的名字

    <current-user>
  <template v-slot:default="slotProps">
    {{ slotProps.user.firstName }}
  </template>
</current-user>

    # 3.作用域插槽的使用 (解构赋值)

    作用域插槽对外提供的数据对象，也可以使用解构赋值简化数据的接收过程 这样可以使模板更简洁，尤其是在该插槽提供了多个 prop的时候

    <current-user v-slot="{ user }">
  {{ user.firstName }}
</current-user>
